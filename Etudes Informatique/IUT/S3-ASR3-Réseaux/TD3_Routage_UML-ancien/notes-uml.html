Notes pour l'utilisation des machines virtuelles Slackware

Auteur : Michel Billaud
Remise à jour : Isabelle Dutour (4/12/2006), M Billaud (22/11/2013)

La commande uml

Cette commande est un script qui regroupe diverses actions sur les machines virtuelle : les créer, les lancer, les arrêter, etc.

pour créer une ou plusieurs machines virtuelles

	uml create slackware nom-de-machine...

Scripts de lancement

  • pour lancer une machine toute seule 
    uml run nom-de-machine
  • pour en lancer plusieurs reliées par un commutateur (switch) virtuel 
    uml run nom1 nom2 ...

Administration et configuration des machines Slackware

Les commandes indiquées ci-dessous sont destinées à être lancées dans la machine virtuelle, par l'administrateur root généralement.

Redémarrage d'une machine virtuelle

Commande reboot ou shutdown -r now (lancée par root).

Arrêt d'une machine virtuelle

Commande halt ou shutdown -h now (lancée par root).

Note : après l'arrêt de la machine virtuelle l'état de la machine virtuelle est sauvegardé sous forme fortement compressée (voir dans ~/.cows), ce qui prend un temps certain.

Éditeurs de textes installés

jed (clone emacs), vi, ed

Les consoles virtuelles

Si on en a trop ou pas assez: éditer le fichier /etc/inittab, et commenter/décommenter les lignes c1 à c6 (garder au moins c0 ...) 

 
#c2:1235:respawn:/sbin/agetty 38400 tty2 linux

Enregistrement des utilisateurs

  • La commande interactive adduser
  • « useradd -m username » ajoute un utilisateur, avec des paramètres par défaut (que l'on peut expliciter). Compléter en utilisant « passwd » pour change le mot de passe, ou « chpasswd » pour les opérations massives.

Modification du nom d'une machine virtuelle

Editer le fichier /etc/HOSTNAME qui contient le nom complet de la machine.

Configuration des interfaces réseau

  • Manuellement on peut faire 
    /sbin/ifconfig ethi adr_ip netmask mask broadcast adr_brdcst up
/sbin/route add default gw adr_gateway
/sbin/route add -net adr_ip_net netmask mask gw adr_gateway
  • Pour une configuration durable, éditer le fichier de configuration /etc/rc.d/rc.inet1.conf + le script /etc/rc.d/rc.inet1 si besoin (route vers une passerelle de sous-réseau par exemple)
  • Un script interactif est aussi disponible : /sbin/netconfig. Il remplit les fichiers
    • /etc/rc.d/rc.inet1.conf (config. interfaces réseau)
    • /etc/HOSTNAME (nom complet de la machine)
    • /etc/hosts (correspondances noms / numéros IP)
    • /etc/resolv.conf (domaines par defaut, serveurs de noms)

Configurer une passerelle

Attention, si on a une machine à 2 interfaces, elle ne joue pas automatiquement le rôle de passerelle : les paquets qui arrivent d'un côté ne sont pas réexpédiés de l'autre. Il faut le demander gentiment à la couche « routage » du noyau, en faisant 

  
echo 1 > /proc/sys/net/ipv4/ip_forward

Sur cette version de la distribution Slackware, pour cela devienne permanent, il suffit de rendre exécutable un script en faisant 

chmod 755 /etc/rc.d/rc.ip_forward

Pour plus d'informations, voir les commentaires du script /etc/rc.d/rc.ip_forward.

Accéder aux fichiers de la machine hôte

L'arborescence de la machine hôte (la vraie, sur laquelle on fait tourner les machines virtuelles) est montée sur /mnt/hosts (voir /etc/fstab)

On retrouve donc ses fichiers dans /mnt/hosts/net/mon-groupe/mon-nom

Installer des packages supplémentaires

Sur la machine hôte, l'image iso du CD ROM d'installation de Linux se trouve dans /net/spare2/CD/slackware-11.0-install-dvd.iso. Pour accéder à l'arborescence de ce CD depuis les machines virtuelles, il suffit de

  • exécuter la commande mount /mnt/dvd : réalise le montage de cette image (voir /etc/fstab),
  • accèder au répertoire /mnt/dvd/slackware.

Les packages sont répartis dans des répertoires par catégorie. Les principales

  • a : système de base (indispensable)
  • ap: applications
  • d: outils de développement
  • e: éditeurs
  • n : networking
Pour chaque package il y a 3 fichiers
  • package.txt : la description
  • package.tgz : l'archive
  • package.tgz.asc : une clé pour authentifier le contenu
Pour chercher quelque chose (par exemple le programme fortune), on peut faire (méthode rustique) 
grep fortune /mnt/dvd/slackware/*/*.txt
On voit alors qu'il s'agit de bsd-games-2.13-i386-1 dans le répertoire y, et alors on installe par 
installpkg /mnt/dvd/slackware/y/bsd-games*.tgz
Il est recommandé d'installer la commande man, qui requiert le package groff, ainsi que les pages de manuel.
